Hybrid Text Summarization System with Comparative Model Analysis

Comparative analysis of a fine-tuned LLaMA model and a custom-built LLM, both trained on social media and news API data

Objective

To create an automated content analysis system that processes large volumes of social media posts and news articles, generating concise summaries and sentiment insights while comparing fine-tuned transformer models against custom architectures for optimal performance.

Design Overview

  • LLaMA + LoRA (PEFT)
  • PyTorch / Transformers
  • Twitter API (Tweepy)
  • Reddit API (PRAW)
  • NewsAPI / GNews
  • FastAPI Backend
  • VADER Sentiment Analysis
  • NLTK / BeautifulSoup

Core Functional Modules

  1. Multi-Source Data Retrieval via Twitter, Reddit, NewsAPI, and GNews APIs
  2. Intelligent Text Processing: dual pipelines for news articles and social media posts
  3. AI-Powered Summarization using fine-tuned LLaMA with LoRA adapters
  4. Sentiment & Trend Analysis with VADER and pattern detection

Model Architecture

Dual-model approach: Fine-tuned LLaMA with parameter-efficient LoRA adapters trained on clean news data then adapted for social media text, compared against a custom-built baseline model for performance benchmarking.

System Pipeline

  • API aggregation layer (Twitter, Reddit, News)
  • Text preprocessing and normalization module
  • LLaMA inference engine with LoRA
  • Custom model for comparative analysis
  • Sentiment analysis and trend detection
  • FastAPI REST backend

Testing & Performance

  • Multi-API rate limiting and error handling
  • Efficient LoRA-based fine-tuning workflow
  • Real-time sentiment scoring accuracy
  • Comparative model evaluation metrics

Evaluation

The system successfully aggregates multi-source data and generates intelligent summaries while demonstrating advanced NLP techniques, transfer learning implementation, and scalable API architecture design.
